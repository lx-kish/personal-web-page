About

Hi! I'm Alex, a software engineer, currently focused on web development and specializing in JavaScript technology stack.

In the area of software development, usability is number one. No one would write a program that will never used. Any software solves a specific set of users’ problems by processing some kind of initial data and it's final representation in a way, which reduces the complexity of the data itself or the representation process. So, as a software engineer, my goal is to provide as many options as possible for as many users as possible, or in other words, to reduce complexity for maximum possible end-users.
To achieve this goal system should be flexible. Flexibility, being the result of an appropriate architecture of a solution, sometimes might conflict with usability, and force end-user to take additional steps in order to get more advantage of further data processing. So, as a software engineer, my goal is to find the balance between usability and flexibility of the solution.
Seeking that balance is the most resource-consuming process in software development, as in many other human activities: as it says, “tapping with a hammer costs $1, knowing where to tap costs $999”. So, as a software engineer, my goal is avoid spending the entire budget on seeking a balance between usability and flexibility of the solution.
This, in a couple of words, is how I build applications.
